Build a full-stack AI project called "Memora: A Persistent and Reconcilable Memory System for LLM Agents".

Goal:
Create a memory layer for AI agents that persists user facts across sessions, detects contradictions, reconciles conflicting information, validates facts, and maintains a full audit trail.

This is not a generic chatbot. The core innovation is a structured memory system that helps an LLM agent remember, update, reconcile, and explain user facts over time.

Important design principles:
1. Not every contradiction should be treated the same.
2. Some facts are stable:
   - birthday
   - dog name
   - family relations
3. Some facts are time-varying:
   - city
   - company
   - job title
   - food preference
4. The system must support memory states:
   - active
   - superseded
   - disputed
   - expired
   - confirmed
5. The system should never silently keep two incompatible active facts for the same stable property.
6. The system should keep history instead of deleting old facts.
7. The system must be explainable: show why a fact was kept, replaced, merged, or marked disputed.

Core system behavior:
When a user sends a message:
1. Extract candidate facts from the message
2. Normalize them into structured memory records
3. Retrieve existing related facts for the same entity/property
4. Detect if the new fact:
   - confirms an existing fact
   - updates an old fact
   - contradicts an existing fact
   - coexists temporally with an old fact
5. Run validation:
   - internal consistency
   - temporal plausibility
   - semantic plausibility
   - optional external verifiability hooks
6. Apply resolution strategy:
   - keep existing
   - replace with new fact
   - keep both with temporal reasoning
   - mark disputed
   - ask user for clarification
7. Save the result with a full audit log
8. Build agent context from resolved active memories only
9. Return an agent response that uses memory correctly

Example scenarios the system must handle:
1. Session 1:
   User: "I work at Google in San Francisco"
   Store:
   - employer = Google
   - city = San Francisco

2. Session 2:
   User: "I just moved to New York for my new job at Meta"
   System should:
   - mark Google as superseded by Meta for employer
   - mark San Francisco as superseded by New York for city
   - preserve history
   - not keep both as active

3. Session 1:
   User: "My dog's name is Max"
   Session 2:
   User: "What's my dog's name?"
   Agent should answer:
   - Max

4. Session 1:
   User: "My birthday is July 15th"
   Session 5:
   User asks birthday
   Agent must never hallucinate another date if memory exists

5. Session 1:
   User: "I hate spicy food"
   Session 2:
   User: "I love spicy food actually"
   System should detect preference reversal and either:
   - update active preference with recency
   - or mark as changed preference over time
